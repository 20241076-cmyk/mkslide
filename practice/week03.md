## 활동내용

## 인상적인 질문과 대답

**Q. 딱딱 끊기지 않고 자연스럽게 색이 변하도록**

색이 자연스럽게 변하려면 선형 보간(lerp)을 사용한다. 현재 색과 다음 색 사이를 시간 비율(t)로 계산해서 중간 색을 만들어낸다.
```python
t = timer / 1000  # 0.0 ~ 1.0 사이 비율
c1 = COLORS[color_index]
c2 = COLORS[(color_index + 1) % len(COLORS)]
blend = (
    int(c1[0] + (c2[0] - c1[0]) * t),
    int(c1[1] + (c2[1] - c1[1]) * t),
    int(c1[2] + (c2[2] - c1[2]) * t)
)
```
예를 들어 빨강→주황 전환 중 t=0.5일 때: R=255, G=63, B=0 → (255, 63, 0) 중간 색이 나온다.

**Q. 이 코드를 초보자한테 알려주듯 설명해줘**

코드는 크게 준비 → 설정 → 반복 세 단계로 이루어져 있다.
- 준비: `pygame`, `math`, `sys` 라이브러리를 불러온다.
- 설정: `pygame.init()`으로 pygame을 켜고 창 크기, 색상, 속도 등을 정의한다.
- 육각형 그리기: 중심에서 360도를 6등분(60도씩)해서 꼭짓점 위치를 `cos`/`sin`으로 계산한 뒤 `pygame.draw.polygon()`으로 그린다.
- 게임 루프: `while True`로 무한 반복하며 매 프레임마다 화면을 지우고 → 입력 처리 → 다시 그리기 → `flip()`으로 표시한다. 마치 영화 필름처럼 빠르게 반복해서 움직이는 것처럼 보이게 만드는 원리다.

**Q. 두 육각형의 시작 위치를 다르게 하자. 화면 중앙을 기준으로 노란색은 왼쪽으로 100픽셀 이동, 파란색은 오른쪽으로 100픽셀 이동, 그리고 두 육각형이 서로 닿으면 화면 왼쪽 상단에 파이썬이라는 문구가 나오도록 수정해줘**

시작 위치는 중앙 좌표에서 각각 100px를 빼고 더해서 설정한다.
```python
yellow_x = WIDTH // 2 - 100
blue_x   = WIDTH // 2 + 100
```
충돌 감지는 피타고라스 정리로 두 중심 사이의 거리를 계산해서 두 반지름의 합보다 작으면 충돌로 판정한다.
```python
distance = math.sqrt((yellow_x - blue_x) ** 2 + (yellow_y - blue_y) ** 2)
is_colliding = distance < HEX_RADIUS * 2
```
