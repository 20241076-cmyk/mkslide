# 4주차 실습 정리

## 오늘 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰

### 원형 vs AABB
원형은 말 그대로 오브젝트 주변에 원을 생성하는 것이며, AABB는 오브젝트 주변에 붉은색의 네모를 추가한다. 원형은 원에 가까운 오브젝트에 히트박스를 넣을 때 유용하며 AABB는 각진 오브젝트에 히트박스를 넣을 때 유용하다.

### AABB vs OBB
AABB는 오브젝트가 회전할 때 따라서 회전하지 않고 오브젝트의 움직임에 따라 새로운 네모를 만드는 방식이라면 OBB는 한 번 만들어진 사각형이 오브젝트의 움직임에 따라 회전하는 방식이라는 점에서 차이가 있었다.

## AI와의 대화에서 배운 것
보통 AABB를 만들 때 사용하는 건 sqrt지만 클로드는 hypot를 사용했다. 이 둘의 차이는 sqrt에선 계산식을 개발자가 직접 기록해야하는 것에 비해 hypot는 계산식이 명령어 안에 들어있기에 계산식 없이 함수만 적어도 해결이 된다는 특징이 있었다.
return rects[0].unionall(rects[1:])라는 코드를 사용하면 비투명 픽셀에 히트박스를 부여함과 동시에 겹치는 히트박스를 하나로 뭉치는 기능이 있다. 해당 기능이 있어야 모든 픽셀을 히트박스 안에 넣는 것과 동시에 빈 공간에 히트박스를 부여하는 걸 최소화할 수 있었다.

## AI에게 물어본 것 중 흥미로웠던 답변

**Q1. 이 코드에서 aabb를 화면에 그리는 부분만 설명해줘. pygame.draw.rect의 인자가 각각 뭘 의미하는지 포함해서 설명해줘**

`pygame.draw.rect(screen, RED, rect, 2)` 이 한 줄이 빨간 AABB 테두리를 그리는 부분이다. 인자는 순서대로 surface(어디에 그릴지), color(색상), rect(위치와 크기), width(선 두께)이며 width가 0이면 꽉 채움, 1 이상이면 테두리만 그린다. width=2이기 때문에 내부는 투명하고 테두리 선만 2px로 그려진다.

---

**Q2. 만약 내가 빨간 사각형을 그냥 사각형보다 3픽셀 정도 크게 만들고 싶다면 rect의 값을 따로 추가해야하는거야?**

pygame에는 `inflate()` 메서드가 있어서 별도의 rect를 만들 수 있다. `aabb_rect = rect.inflate(6, 6)`처럼 사용하면 중심을 기준으로 좌우 3px, 상하 3px씩 늘어난 새로운 rect가 반환된다. 3px 키우려면 6을 넣는 이유는 inflate가 양쪽에 나눠서 적용하기 때문이다. 원본 rect는 변하지 않는다.

---

**Q4. math.sqrt를 썼어? 안 썼다면 안 쓴 이유와 sqrt를 쓰는 이유를 설명해줘**

`math.sqrt`는 쓰지 않고 대신 `math.hypot`를 사용했다. sqrt로 직접 구현하면 `math.sqrt((cx_a - cx_b)**2 + (cy_a - cy_b)**2)`처럼 공식을 풀어써야 하지만 hypot는 `math.hypot(dx, dy)` 한 줄로 동일한 결과가 나온다. 계산량은 동일하고 코드가 짧고 의도가 명확해서 hypot를 선택했다.

---

**Q10. return rects[0].unionall(rects[1:]) 이 코드가 없으면 어떤 일이 일어나지?**

`get_bounding_rects()`는 비투명 픽셀이 연결된 덩어리마다 하나씩 rect를 반환한다. `unionall`이 없이 `rects[0]`만 반환하면 첫 번째 덩어리만 반환되어 나머지 덩어리가 BB에서 누락된다. 반환값이 아예 없으면 이후 코드에서 None에 접근하려다 에러가 발생한다. `unionall`이 있어야 모든 덩어리를 하나의 rect로 합쳐서 오브젝트를 빠짐없이 포함할 수 있다.
